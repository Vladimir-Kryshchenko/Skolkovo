# CLAUDE.md — База Сколково

Главный гайд проекта для Claude Code. Читается в начале каждой сессии.

## Что это за проект

Сервис «База Сколково» — RAG-база знаний по документам Фонда «Сколково» с открытым MCP-доступом для агентов и приложений, **системой управления резидентством** и **Telegram-ботом** для клиентов.

**Ключевые подсистемы:**
1. **Парсер (`src/scraper/`)** — заводит каталог документов из RSS-ленты dochub.sk.ru (см. ниже про WAF).
2. **Новости (`src/news/`)** — RSS-мониторинг новостей/событий, попадают в RAG как документы категории «Новости».
3. **Мероприятия (`src/events/`)** — парсинг мероприятий через RSS и HTML-скрапинг, индексация в RAG.
4. **Конкурсы (`src/contests/`)** — парсинг конкурсов/грантов с HTML-страниц, индексация в RAG.
5. **FAQ (`src/faq/`)** — парсинг FAQ-страниц, индексация в RAG. Рядом: **Резиденты (`src/residents/`)** — парсинг реестра участников, наполняет `search_residents` (команда `residents`, `RESIDENTS_ENABLED`).
6. **Telegram (`src/telegram/`)** — мониторинг Telegram-каналов через RSS-агрегаторы (rsshub.app).
7. **RAG-сервис (`src/rag_service/`)** — chunking, эмбеддинги, индексация в Qdrant, поиск с фильтром «действует».
8. **MCP-сервер (`src/mcp_server/`)** — **открытый** MCP-эндпоинт (API-ключ, rate-limit по IP); более 25 инструментов (документы, источники, резидентство, лента изменений, генерация документов, агенты).
9. **Админка (`src/admin/`)** — управление документами, классификация, валидация, версионирование, статистика, граф связей, diff сравнение. **Порт: :8090.**
10. **ИИ Модели и Агенты (`src/admin/ai_admin.go`)** — настройка LLM-провайдеров (Alibaba Qwen, OpenAI, Anthropic, Custom) и агентов (Консультант, Валидатор, Монитор, Координатор, Аннотатор страниц). Страницы `/ai/models` и `/ai/agents` внутри Админки (:8090).
11. **Резидентство-Админ (`src/admin/residency_admin.go`)** — управление клиентами, стадиями, чек-листами, дедлайнами, шаблонами, тенантами, мероприятиями, конкурсами. **Порт: :8091.**
12. **Портал клиента (`src/portal/`)** — Личный кабинет резидента: дашборд, чек-листы, дедлайны, документы, генерация. Вход через magic-link по email (без пароля). **Порт: :8092.**
13. **Чат-виджет (`src/chat_widget/`)** — встраиваемый JS-виджет и standalone чат-страница. Проксирует запросы в MCP-сервер. **Порт: :8093.**
14. **Дашборд консультанта (`src/admin/consultant.go`)** — сводка по клиентам для менеджеров: просроченные/критичные/предстоящие дедлайны, прогресс стадий. **Порт: :8094.**
15. **Резидентство (`src/common/store/`, `src/mcp_server/residency_tools.go`)** — клиенты, стадии, чек-листы, дедлайны, шаблоны.
16. **Telegram-бот (`src/tgbot/`)** — бот для клиентов резидентства: /start, /status, /deadlines, /docs, /ask, /help.
17. **Аналитика (`src/analytics/`)** — сбор метрик, HTML-дашборд с Chart.js, экспорт в CSV.
18. **Diff (`src/diff/`)** — сравнение версий документов с LCS-алгоритмом, генерация HTML.
19. **Validator (`src/validator/`)** — проверка полноты базы, отчёты в Markdown/HTML.
20. **Migrate (`src/migrate/`)** — миграции схемы PostgreSQL.
21. **Конвейер (`src/pipeline/`)** + уведомления (`src/notify/`) — регулярная актуализация по расписанию.
22. **Лента изменений (`src/changes/`)** — фиксирует, что появилось/обновилось/устарело по всем источникам; питает `get_recent_changes` и Telegram-алерты консультанту.
23. **Мониторинг свежести (`src/health/`)** — SLA-состояние каждого источника (когда успешно обновлялся, сколько вернул, не падает ли); `get_source_health`.
24. **Аудит охвата (`src/audit/`)** — сверяет канонический каталог источников с реально настроенными/наполненными; команда `audit`, `get_coverage_audit`.
25. **Генератор документов (`src/generator/`)** — заполняет шаблоны данными клиента, выдаёт PDF (через chromedp) и DOCX; команда `generate`, `generate_document`.
26. **Навигация по сайту (`src/navindex/`)** — специализированная RAG-база структуры всех 6 интерфейсов (страница → вкладка → блок → надпись → «как попасть»). Источник истины — `tree.go`; команда `navindex` (render/export/index) генерирует `RAG_Структура_сайта.md`, `navigation.json` и отдельную Qdrant-коллекцию `skolkovo_navigation`. Питает MCP-инструмент `get_navigation` для чат-бота. Переиндексируется при старте `serve`.
27. **Страницы сайта (`src/sitepages/`)** — отдельный от файлов-документов слой: краулер обходит публичные страницы сайта (`SITEPAGES_SEEDS`: sk.ru/dochub) и хранит **по одной записи на страницу** (URL, заголовок, краткое описание, раздел, хэш контента) в таблице `site_pages` и в отдельной Qdrant-коллекции `skolkovo_sitepages`. **ИИ-обогащение (`enrich.go`, миграция 010):** каждая страница автоматически аннотируется агентом «Аннотатор страниц» (`page_annotator` в `/ai/agents`) — генерируются **теги** (гибридный растущий словарь `site_page_tags`), **краткое описание**, **цели**, **важные тезисы**, **выводы**; эти поля идут в RAG-эмбеддинг и payload. Команда `sitepages` (crawl/**enrich**[`--all`]/index/all), периодический обход + аннотирование в `serve`, единоразовый бэкфилл всей базы при старте (`SITEPAGES_ENRICH_ENABLED`); переаннотирование — только при изменении контента (`enrich_hash`). В админке `/sitepages`: фильтр по тегам (мультиселект), в просмотрщике — ИИ-секции и связанные страницы (по общим тегам + семантически близкие). Изменения попадают в ленту (`entity_type=sitepage`), а свежесть — в `get_source_health` (источник `sitepages`). Питает MCP-инструмент `search_site_pages`. **Не путать:** `search_documents` — текст файлов-документов, `get_navigation` — устройство внутренних интерфейсов, `search_site_pages` — страницы публичного сайта.

**Важно про источник:** dochub.sk.ru — Telligent Evolution; список грузится JS, поэтому каталог берётся из RSS (`…/foundation/documents/rss.aspx`). Страницы файлов `/m/docs/` за **WAF (403)** — тела файлов автоскачиванием не берутся (chromedp подключён для headless-браузера). Детали — в архитектурном документе, раздел 7.

## Главная ценность

Агенты и приложения через MCP всегда получают **актуальную** информацию: видно, какие документы действуют, а какие устарели. Парсер поддерживает базу в актуальном состоянии. Система резидентства управляет жизненным циклом клиентов от заявки до выхода.

## Правила проекта (обязательны)

- **Схемы в `.md` — только Mermaid** (см. `Правила_проекта/ПРАВИЛО_Схемы_в_Markdown_Mermaid_v1_0_29_05_2026.md`). ASCII-арт запрещён.
- **Структура папок** — см. `Правила_проекта/ПРАВИЛО_Структура_папок_v1_0_29_05_2026.md`. Первоисточники в `Документы_Сколково/`, код в `src/`, данные в `data/`.
- **Именование документов:** `ТИП_Название_vМ_м_ДД_ММ_ГГГГ.md`.
- **Даты — абсолютные.**

## Жизненный цикл документа

`На_проверке` → (валидация в админке) → `Действует` → (вышла новая редакция / отмена) → `Устарел/Архив`.
Статус и метаданные ведутся в `Документы_Сколково/Метаданные/`.

## Стадии резидентства клиента (8 стадий)

`Подача_заявки` → `Экспертиза` → `Решение` → `Договор` → `Резидент` → `Отчётность` → `Продление` → `Выход`.
Переходы между стадиями контролируются, история переходов сохраняется. Каждая стадия имеет свои чек-листы и дедлайны.
Чек-листы определены для процедур: Вступление, Отчётность, Продление, Выход.

## Технологический стек

- **Бэкенд: Go 1.26** — единый бинарь, запускает 6 HTTP-серверов: MCP (:8080), Админ (:8090), Резидентство-Админ (:8091), Портал клиента (:8092), Чат-виджет (:8093), Дашборд консультанта (:8094). Метрики Prometheus на :9090.
- **Векторная БД: Qdrant** (Docker, фильтрация по метаданным — отдаём только `действует`).
- **Эмбеддинги: локальные multilingual-e5** через HuggingFace **TEI** (Text Embeddings Inference) в Docker; Go обращается по HTTP.
- **MCP: открытый + API-ключ**, транспорт Streamable HTTP; rate-limit по IP.
- **Реестр метаданных и резидентства:** Postgres (статусы, версии, классификация, клиенты, чек-листы, дедлайны, AI-конфиг моделей и агентов) — он же БД всех подсистем.
- **Headless-браузер: chromedp** — для обхода WAF при скачивании тел файлов и генерации PDF.
- **Telegram Bot API:** go-telegram-bot-api/v5 — бот для клиентов резидентства.
- **Мониторинг (prod):** Prometheus + Grafana в отдельных контейнерах, nginx как reverse proxy с TLS.
- **Развёртывание:** docker-compose (Qdrant + TEI + Postgres + Go-сервис). При старте `serve` миграции БД применяются автоматически (`AUTO_MIGRATE=true`, идемпотентно); образ включает каталог `migrations/`.

Подробности и схемы — `Аналитика/Документация/Техническая_документация/Архитектура/`.

## MCP-инструменты

### Базовые (документы)
| Инструмент | Описание |
| :--- | :--- |
| `search_documents` | Семантический поиск по действующим документам |
| `get_document` | Метаданные документа по ID |
| `list_active_documents` | Список действующих документов (опц. по категории) |

### Расширенные источники
| Инструмент | Описание |
| :--- | :--- |
| `search_events` | Поиск мероприятий (query, date_from, date_to, category) |
| `get_event` | Мероприятие по ID |
| `search_contests` | Поиск конкурсов/грантов (query, status, category) |
| `get_contest` | Конкурс/грант по ID |
| `search_faq` | Поиск по FAQ (query, category) |
| `search_residents` | Поиск резидентов (query, industry, status) |
| `get_document_links` | Связи документа (document_id, link_type) |

### Льготы и НПА
| Инструмент | Описание |
| :--- | :--- |
| `search_preferences` | Поиск льгот (query, pref_type: tax_profit/insurance/vat/customs/other, status: active/outdated, limit) |
| `get_preference` | Льгота по идентификатору (preference_id) |
| `search_npa` | Поиск нормативно-правовых актов (query, npa_type: law/decree/order/decision, status: active/amended/revoked, limit) |
| `get_npa` | НПА по идентификатору (npa_id) |

### Актуальность и мониторинг
| Инструмент | Описание |
| :--- | :--- |
| `get_recent_changes` | Лента изменений базы: что появилось/обновилось/устарело (since_days, entity_type, category, limit) |
| `get_source_health` | SLA-состояние источников: когда последний раз успешно обновлялись, не «протухли» ли |
| `get_coverage_audit` | Полнота охвата источников: что покрыто, а что не настроено/устарело/без данных |

### Навигация по сайту (для чат-бота)
| Инструмент | Описание |
| :--- | :--- |
| `get_navigation` | Где на сайте нужная функция/страница/вкладка/блок и как туда попасть (query, limit). Отдельная Qdrant-коллекция навигации (`skolkovo_navigation`), источник истины — `src/navindex/tree.go`. Не путать с `search_documents` (содержание документов). |
| `search_site_pages` | Поиск по **страницам публичного сайта** Сколково (sk.ru/dochub): одна запись на страницу — заголовок, ИИ-описание, авто-теги, раздел, URL (query, **tags** через запятую, limit). Отдельная коллекция `skolkovo_sitepages` (`src/sitepages/`). Не путать с `search_documents` (файлы-документы) и `get_navigation` (внутренние интерфейсы). |

### Документы клиента (генерация)
| Инструмент | Описание |
| :--- | :--- |

| `generate_document` | Готовый файл документа (PDF/DOCX) из шаблона по данным клиента (template_id, client_id, variables) |
| `list_document_templates` | Список доступных шаблонов для генерации |

### ИИ-агенты (через MCP)
| Инструмент | Описание |
| :--- | :--- |

| `validate_document` | Проверка документа против чек-листа процедуры (document_text, procedure_type: entry/reporting/extension/exit, опц. client_id) |

| `check_eligibility` | Проверка квалификации компании по ИНН (inn) |
| `get_document_file` | Скачать файл документа (base64, до 8 МБ) (id) |
