# CLAUDE.md — База Сколково

Главный гайд проекта для Claude Code. Читается в начале каждой сессии.

## Что это за проект

Сервис «База Сколково» — RAG-база знаний по документам Фонда «Сколково» с открытым MCP-доступом для агентов и приложений.

**Ключевые подсистемы:**
1. **Парсер (`src/scraper/`)** — заводит каталог документов из RSS-ленты dochub.sk.ru (см. ниже про WAF).
2. **Новости (`src/news/`)** — RSS-мониторинг новостей/событий, попадают в RAG как документы категории «Новости».
3. **RAG-сервис (`src/rag_service/`)** — chunking, эмбеддинги, индексация в Qdrant, поиск с фильтром «действует».
4. **MCP-сервер (`src/mcp_server/`)** — **открытый** MCP-эндпоинт (API-ключ, rate-limit по IP); инструменты поиска и получения документов.
5. **Админка (`src/admin/`)** — управление, классификация, валидация, версионирование, статистика.
6. **Конвейер (`src/pipeline/`)** + уведомления (`src/notify/`) — регулярная актуализация по расписанию.

**Важно про источник:** dochub.sk.ru — Telligent Evolution; список грузится JS, поэтому каталог берётся из RSS (`…/foundation/documents/rss.aspx`). Страницы файлов `/m/docs/` за **WAF (403)** — тела файлов автоскачиванием не берутся (нужен headless-браузер/доступ). Детали — в архитектурном документе, раздел 7.

## Главная ценность

Агенты и приложения через MCP всегда получают **актуальную** информацию: видно, какие документы действуют, а какие устарели. Парсер поддерживает базу в актуальном состоянии.

## Правила проекта (обязательны)

- **Схемы в `.md` — только Mermaid** (см. `Правила_проекта/ПРАВИЛО_Схемы_в_Markdown_Mermaid_v1_0_29_05_2026.md`). ASCII-арт запрещён.
- **Структура папок** — см. `Правила_проекта/ПРАВИЛО_Структура_папок_v1_0_29_05_2026.md`. Первоисточники в `Документы_Сколково/`, код в `src/`, данные в `data/`.
- **Именование документов:** `ТИП_Название_vМ_м_ДД_ММ_ГГГГ.md`.
- **Даты — абсолютные.**

## Жизненный цикл документа

`На_проверке` → (валидация в админке) → `Действует` → (вышла новая редакция / отмена) → `Устарел/Архив`.
Статус и метаданные ведутся в `Документы_Сколково/Метаданные/`.

## Технологический стек

- **Бэкенд: Go 1.25** — единый бинарь с подсистемами scraper / rag / mcp / admin (или раздельные сервисы).
- **Векторная БД: Qdrant** (Docker, фильтрация по метаданным — отдаём только `действует`).
- **Эмбеддинги: локальные multilingual-e5** через HuggingFace **TEI** (Text Embeddings Inference) в Docker; Go обращается по HTTP.
- **MCP: открытый + API-ключ**, транспорт Streamable HTTP; rate-limit.
- **Реестр метаданных:** Postgres (статусы, версии, классификация) — он же БД админки.
- **Развёртывание:** docker-compose (Qdrant + TEI + Postgres + Go-сервис).

Подробности и схемы — `Аналитика/Документация/Техническая_документация/Архитектура/`.
