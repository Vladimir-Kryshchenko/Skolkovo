## 1. Реальная карта серверов (7 портов)

> **Критическое расхождение с CLAUDE.md**: документ упоминает только `:8080` и `:8090`.  
> Реально запускается 7 HTTP-серверов.

| Порт | Сервис | Аудитория | Аутентификация |
|------|--------|-----------|----------------|
| **:8080** | MCP-сервер | ИИ-агенты, внешние приложения | API-ключ (X-API-Key) + rate-limit по IP |
| **:8090** | Админ-панель (документы + AI) | Администраторы | HTTP Basic Auth |
| **:8091** | Резидентство-Админ | Менеджеры резидентства | HTTP Basic Auth |
| **:8092** | Портал (Личный кабинет клиента) | Клиенты-резиденты | Magic-link по email (без пароля) |
| **:8093** | Чат-виджет | Публичные пользователи | Нет (публичный) |
| **:8094** | Дашборд консультанта | Консультанты | Без авторизации (внутренний) |
| **:9090** | Prometheus /metrics | Мониторинг | Нет (внутренний) |

### 5.4 Потенциальная проблема навигации

Резидентство-Админ (:8091) и Дашборд консультанта (:8094) — **без явной аутентификации** в коде (нужно проверить). Если они не защищены, это уязвимость.

### 5.5 Проблема UX (по замечанию заказчика)

> «ИИ агенты усложняют работу пользователя»

Наблюдение: В системе сейчас **4 точки контакта с ИИ**:
1. MCP (:8080) — для агентов-разработчиков
2. Чат-виджет (:8093) — для конечных пользователей
3. AI-консультант в агентах — настраивается через (:8090)
4. Прямые LLM-вызовы через агентов

Рекомендация: Чат-виджет (:8093) должен быть **единственной точкой входа для пользователя**. Агент «Консультант» должен незаметно решать задачи через MCP-инструменты — пользователь пишет в чат и получает ответ, не зная о внутренних системах.
