# RAG: Полная структура сайта «База Сколково»

> Навигационная карта для ИИ-чат-бота: каждая страница, вкладка, блок и надпись.
> Сгенерировано из исходного кода пакетом `src/navindex` (источник истины — `Tree()`).
> Не редактируйте вручную: при изменении страниц правьте `src/navindex/tree.go` и пересоберите командой `navindex`.

## Карта интерфейсов

| Порт | Интерфейс | Для кого | Как попасть |
| :--- | :--- | :--- | :--- |
| :8092 | Личный кабинет клиента (Портал) | Клиенты-резиденты, компании, подающие заявку на резидентство Сколково | Открыть ссылку из письма (magic-link) или зайти на :8092 и запросить ссылку по email. |
| :8093 | Чат-виджет «Консультант Сколково» | Все пользователи сайта | Кнопка-пузырь «Консультант» внизу справа на любой странице, либо отдельная страница :8093/chat. |
| :8094 | Дашборд консультанта | Менеджеры резидентства | Открыть :8094/consultant/dashboard, ввести логин/пароль консультанта. |
| :8091 | Резидентство-Админ | Менеджеры резидентства | Открыть :8091/clients. |
| :8090 | Главная Админ-панель | Администраторы | Открыть :8090, войти логином/паролем. |
| :8080 | MCP-сервер (открытый API для ИИ-агентов) | ИИ-агенты, приложения, разработчики | POST :8080/mcp с заголовком Authorization: Bearer <ключ>. /health открыт. |
| Telegram | Telegram-бот для клиентов резидентства | Клиенты-резиденты, менеджеры | Найти бота в Telegram, нажать Start или написать /start. Email должен совпадать с email клиента в системе. |

## Личный кабинет клиента (Портал) (:8092)

- **Аудитория:** Клиенты-резиденты, компании, подающие заявку на резидентство Сколково
- **Аутентификация:** Magic-link на email (без пароля). Сессия 24 часа, токен ссылки 15 минут.
- **Как попасть:** Открыть ссылку из письма (magic-link) или зайти на :8092 и запросить ссылку по email.

### Вход в личный кабинет — `/login`

**Назначение:** Запросить ссылку для входа без пароля.  
**Как попасть:** Открывается автоматически, если не авторизован.

- **Форма входа** _(form)_ — Запрос magic-link по email.
  - Элементы: Логотип «🏛 База Сколково»; Личный кабинет резидента; Электронная почта (Email); поле name@company.ru; кнопка «Отправить ссылку для входа»; переключатель темы
- **Сообщения** _(section)_
  - Элементы: успех (зелёный); ошибка (красный)

### Дашборд — `/dashboard`

**Назначение:** Обзор текущей стадии, дедлайнов, прогресса и новостей базы.  
**Как попасть:** Пункт меню «Дашборд» (первый в навигации) или вход по ссылке.

- **Шапка (общая для всех страниц ЛК)** _(nav)_ — Верхняя шапка кабинета.
  - Элементы: логотип «База Сколково»; email текущего клиента (data-tooltip «Текущий аккаунт»); переключатель темы; кнопка «Выйти» (ведёт на /logout)
- **Навигационное меню (общее для всех страниц ЛК)** _(nav)_ — Главное меню кабинета; пункты идут в этом порядке.
  - Элементы: Дашборд (/dashboard); Чек-листы (/checklists); Дедлайны (/deadlines); Документы (/documents); Подписки (/subscriptions); Генерация (/generate)
- **Баннер-пояснение** _(section)_ — Разграничение «нашего сопровождения» и официального кабинета Сколково.
  - Элементы: «Это ваш прогресс в нашем сопровождении»; ссылка «личный кабинет Сколково cabinet.sk.ru»
- **Текущая стадия** _(section)_ — Стадия резидентства и прогресс по стадиям.
  - Элементы: бейдж стадии; progress bar; процент
- **Прогресс чек-листов** _(section)_
  - Элементы: название чек-листа; процент завершения; progress bar; пусто: «Чек-листы пока не назначены»
- **Ближайшие дедлайны** _(table)_
  - Элементы: Дедлайн; Дата; Статус; пусто: «Нет ближайших дедлайнов»
- **Документы сопровождения** _(section)_
  - Элементы: бейджи pending/submitted/approved/rejected; пусто: «Документы пока не назначены»; примечание: это документы в рамках сопровождения, не официального кабинета Сколково
- **Что нового в базе Сколково** _(widget)_ — Лента изменений базы за 7 дней: новые/обновлённые/устаревшие сущности.
  - Элементы: бейдж Новая/Обновлена/Устарела/Удалена; тип сущности; дата; заголовок; описание
- **Telegram-бот (если настроен)** _(widget)_ — Кнопка открытия бота для быстрого доступа к статусу и дедлайнам.
  - Элементы: название бота (@username); описание возможностей; кнопка «Открыть @BotUsername»

### Мои чек-листы — `/checklists`

**Назначение:** Список шагов для прохождения процедур резидентства.  
**Как попасть:** Пункт меню «Чек-листы».

- **Карточки чек-листов** _(section)_
  - Элементы: название; тип процедуры; статус; progress bar; X/Y шагов; шаги: выполнен/в процессе/не начат; пусто: «Чек-листы пока не назначены»

### Дедлайны — `/deadlines`

**Назначение:** Сроки подачи документов и отчётности.  
**Как попасть:** Пункт меню «Дедлайны».

- **Ближайшие дедлайны** _(table)_
  - Элементы: Название; Дата; Тип; Статус; completed/overdue/upcoming
- **Просроченные** _(table)_
  - Элементы: Название; Дата; Статус; бейдж «Просрочен»; пусто: «Нет просроченных дедлайнов»

### Мои документы — `/documents`

**Назначение:** Документы, связанные с резидентством клиента.  
**Как попасть:** Пункт меню «Документы».

- **Таблица документов** _(table)_
  - Элементы: Документ; Роль; Статус; Дата; Источник; пусто: «Документы пока не назначены»

### Генерация документа — `/generate`

**Назначение:** Создать документ из шаблона по данным профиля (PDF/DOCX).  
**Как попасть:** Пункт меню «Генерация» (последний в навигации).

- **Генерация документа из шаблона** _(form)_
  - Элементы: Шаблон документа (выпадающий список); — Выберите шаблон —; кнопка «Сгенерировать»; подсказка: «Выберите шаблон — документ будет создан автоматически на основе данных вашего профиля и сохранён в раздел «Документы».»
- **Доступные шаблоны** _(section)_
  - Элементы: карточки шаблонов (название, тип, версия); карточка кликабельна — клик генерирует документ по шаблону; пусто: «Шаблоны пока не добавлены» + «Шаблоны документов создаются администратором в разделе резидентства»

### Уведомления — `/notifications`

**Назначение:** Inbox персональных уведомлений об изменениях документов, важных для стадии клиента.  
**Как попасть:** Прямой URL :8092/notifications. В навигационном меню пункта нет (иконки-колокольчика тоже); на странице «Подписки» есть пояснение, что уведомления «появляются в разделе «Уведомления»».

- **Шапка (своя, вне общего layout)** _(nav)_
  - Элементы: заголовок «Уведомления»; ссылка «Личный кабинет» (на /dashboard); переключатель темы
- **Список уведомлений** _(section)_
  - Элементы: Непрочитанные: X из Y; бейдж Критично/Важно/Информация; заголовок; дата; описание; ссылка «Источник»; кнопка «Отметить прочитанным»; пусто: «Пока нет уведомлений об изменениях»

### Подписки на уведомления — `/subscriptions`

**Назначение:** Управление подписками клиента на категории изменений базы знаний Сколково.  
**Как попасть:** Пункт меню «Подписки».

- **Пояснение** _(section)_
  - Элементы: «Выберите категории изменений, о которых хотите получать уведомления. Уведомления появляются в разделе «Уведомления» и могут приходить в Telegram-боте.»
- **Список категорий подписок** _(section)_
  - Элементы: Нормативные документы — Изменения в НПА, льготах и регламентах Сколково; Мероприятия — Новые и изменённые мероприятия Сколково; Конкурсы и гранты — Новые конкурсы, изменения статусов, итоги; Отчётность — Изменения в требованиях к отчётности резидентов; чекбоксы выбора категорий; кнопка «Сохранить подписки»; сообщение «Подписки обновлены: N категорий»

## Чат-виджет «Консультант Сколково» (:8093)

- **Аудитория:** Все пользователи сайта
- **Аутентификация:** Публичный (без авторизации).
- **Как попасть:** Кнопка-пузырь «Консультант» внизу справа на любой странице, либо отдельная страница :8093/chat.

### Чат с консультантом (standalone) — `/chat`

**Назначение:** Полноэкранный чат с ИИ-консультантом по базе Сколково.  
**Как попасть:** Открыть :8093/chat.

- **Шапка** _(nav)_
  - Элементы: Консультант Сколково; переключатель темы (tooltip «Переключить тему»); заголовок вкладки «Чат — Сколково»
- **Область сообщений** _(section)_
  - Элементы: приветствие «Здравствуйте! Чем могу помочь?»; сообщения пользователя/ассистента; Markdown; индикатор «Печатает…»; ошибка соединения «Ошибка соединения.»
- **Поле ввода** _(form)_
  - Элементы: Введите сообщение… (tooltip «Нажмите Enter для отправки»); Enter — отправить, Shift+Enter — новая строка; кнопка «Отправить» (tooltip «Отправить сообщение»)

### API: сессия чата (POST /api/session) — `/api/session`

**Назначение:** Создаёт/возвращает сессию чата (JSON {id}). Внутренний эндпоинт виджета.  
**Как попасть:** Виджет вызывает автоматически при инициализации.

- **Ответ** _(section)_
  - Элементы: id сессии

### API: сообщение чата (POST /api/chat) — `/api/chat`

**Назначение:** Проксирует сообщение пользователя в MCP-инструмент ask_consultant и возвращает ответ консультанта.  
**Как попасть:** Виджет вызывает при отправке сообщения.

- **Обработка** _(section)_
  - Элементы: вызов MCP ask_consultant; ответ ассистента с источниками

### Встраиваемый виджет — `/chat-widget.js`

**Назначение:** Плавающая кнопка и панель чата, встраивается на сторонний сайт через <script>.  
**Как попасть:** Подключить <script src=".../chat-widget.js" data-skolkovo-chat='{...}'>.

- **Плавающая кнопка** _(button)_
  - Элементы: иконка чат-пузыря, низ-право
- **Панель чата** _(widget)_
  - Элементы: шапка «Консультант»; сообщения; поле «Сообщение…»; кнопка «Отправить»; кнопка закрытия

## Дашборд консультанта (:8094)

- **Аудитория:** Менеджеры резидентства
- **Аутентификация:** HTTP Basic Auth (логин/пароль). Переменные: CONSULTANT_USER, CONSULTANT_PASS. Если пусты — доступ открыт по прямому URL.
- **Как попасть:** Открыть :8094/consultant/dashboard, ввести логин/пароль консультанта.

### Дашборд консультанта — `/consultant/dashboard`

**Назначение:** Сводка по клиентам по срочности дедлайнов и важным изменениям документации.  
**Как попасть:** Прямой URL :8094/consultant/dashboard.

- **Шапка** _(nav)_ — Верхняя панель дашборда.
  - Элементы: заголовок «Дашборд консультанта»; время последнего обновления; ссылка «Документы» (возврат на главную админку /); переключатель темы
- **Сводные карточки** _(section)_
  - Элементы: ПРОСРОЧЕНО; КРИТИЧНО (≤3 ДН.); ВНИМАНИЕ (≤7 ДН.); В ПОРЯДКЕ; ВСЕГО КЛИЕНТОВ
- **Важные изменения документации (30 дней)** _(section)_
  - Элементы: бейдж Критично/Важно/Инфо; заголовок события; дата; префикс «Стадии: …» (затронутые стадии)
- **Таблица клиентов** _(table)_
  - Элементы: Клиент / ИНН (имя — ссылка; префикс «ИНН: …»; страница карточки /consultant/client/{id} пока не реализована); Стадия (буквенный бейдж — первая буква стадии); В стадии: «N дн.»; Прогресс (%); Ближайший дедлайн: «через N дн.» / «просрочен на N дн.» / «—»; Статус: значения бейджа «Просрочен» / «Критично» / «Внимание» / «В порядке»; сортировка: просрочено→критично→внимание→в порядке; пусто: «Клиенты не найдены»

## Резидентство-Админ (:8091)

- **Аудитория:** Менеджеры резидентства
- **Аутентификация:** HTTP Basic Auth (логин/пароль). Кнопки выхода нет — сессия завершается закрытием браузера.
- **Как попасть:** Открыть :8091/clients.

### Клиенты — Резидентство — `/clients`

**Назначение:** Список клиентов с фильтром по стадиям и поиском.  
**Как попасть:** Пункт меню «Клиенты». Сайдбар (группы): «Резиденты» — Клиенты, Чек-листы (= шаблоны чек-листов процедур, не клиентские), Дедлайны; «Каталоги» — Шаблоны, Тенанты, Мероприятия, Конкурсы; «Настройки» — ИИ-модели (ведёт на общую ИИ-конфигурацию :8090/ai/models). Кнопки «Выход» нет (HTTP Basic Auth).

- **Счётчики по стадиям** _(section)_ — Stat-карточки над фильтрами.
  - Элементы: Всего; по одной карточке на стадию (Подача заявки, Экспертиза, Решение, Договор, Резидент, Отчётность, Продление, Выход) с числом клиентов
- **Фильтры** _(section)_
  - Элементы: стадии: Подача заявки, Экспертиза, Решение, Договор, Резидент, Отчётность, Продление, Выход; поиск (плейсхолдер «Поиск по ИНН или имени…»)
- **Таблица клиентов** _(table)_
  - Элементы: Название / ИНН; Email; Стадия; Тенант; Обновлён; колонка «Действия» с кнопкой «Карточка»; пусто: «Нет клиентов» / «Клиенты появятся после подачи заявки на резидентство»

## Главная Админ-панель (:8090)

- **Аудитория:** Администраторы
- **Аутентификация:** HTTP Basic Auth (логин/пароль). Вход только для администратора (ADMIN_USER/ADMIN_PASSWORD); остальные пользователи работают с базой через MCP, Telegram-бот и ИИ-агентов.
- **Как попасть:** Открыть :8090, войти логином/паролем.

### Документы — `/`

**Назначение:** Каталог документов базы: статусы, категории, файлы, индексация; сбор данных, индексация и модерация.  
**Как попасть:** Пункт меню «Документы» (главная). Сайдбар (группы): «База знаний» — Документы (/), Изменения (/changes), Страницы сайта (/sitepages), Сравнение версий (/diff), Граф связей (/graph), Аналитика (/analytics); «Источники и ИИ» — ИИ-модели (/ai/models), ИИ-агенты (/ai/agents), Прокси / VPN (/proxy), Льготы и НПА (/regulations). Внизу: переключатель темы, кнопка «Выход».

- **Карточки статистики** _(section)_
  - Элементы: Всего; Активные; На проверке; Устарелые; Архив; Отклонённые; Индексировано
- **Панель операций** _(section)_ — Три группы кнопок управления базой.
  - Элементы: группа «Сбор данных»: «Обновить из источников», «Скачать файлы», «Локальные файлы»; группа «Индексация»: «Переиндексировать поиск», «Полное обновление», «Карта навигации»; группа «Модерация»: «✓ Одобрить все (N)»
- **Фильтр по статусу** _(tab)_ — Вкладки-фильтры по статусу документа.
  - Элементы: Все; Действует; На проверке; Устарел; Архив; Отклонён
- **Фильтры и поиск** _(form)_
  - Элементы: поиск по заголовку; фильтр категории
- **Таблица документов** _(table)_
  - Элементы: Заголовок; Категория; Статус; Файл; действия: смотреть оригинал / «Текст для поиска» (чанки) / скачать / загрузить файл / версия (заменяет) / снять с индекса / удалить / сменить статус / сменить категорию

### Страницы сайта — `/sitepages`

**Назначение:** Каталог публичных страниц сайта Сколково (sk.ru/dochub) с ИИ-аннотациями и авто-тегами; одна запись на страницу.  
**Как попасть:** Пункт меню «Страницы сайта» (группа «База знаний»).

- **Инфо-бар** _(section)_
  - Элементы: «Последний обход сайта»; «Показано страниц: N»; пусто: «Хранилище страниц сайта не подключено»
- **Фильтры** _(form)_
  - Элементы: Поиск (плейсхолдер «По заголовку, URL, разделу…»); Раздел (Все разделы / список); Статус (Все / Доступна / Недоступна); Теги (мультиселект: «Все теги» / «Выбрано тегов N», поиск тегов, «страница содержит все выбранные»); Изменено от; Изменено до; кнопка «Применить»; кнопка «Сбросить»

## MCP-сервер (открытый API для ИИ-агентов) (:8080)

- **Аудитория:** ИИ-агенты, приложения, разработчики
- **Аутентификация:** API-ключ (Bearer) + rate-limit по IP. Транспорт Streamable HTTP.
- **Как попасть:** POST :8080/mcp с заголовком Authorization: Bearer <ключ>. /health открыт.

### MCP-инструменты — `/mcp`

**Назначение:** Более 35 инструментов: документы, источники, льготы/НПА, резидентство, изменения, генерация, агенты, навигация, страницы сайта.  
**Как попасть:** Вызов через tools/call.

- **Документы** _(section)_
  - Элементы: search_documents; get_document; list_active_documents; get_document_file
- **Источники** _(section)_
  - Элементы: search_events; get_event; search_contests; get_contest; search_faq; search_residents; get_document_links
- **Льготы и НПА** _(section)_
  - Элементы: search_preferences; get_preference; search_npa; get_npa
- **Актуальность и мониторинг** _(section)_
  - Элементы: get_recent_changes (params: since_days, entity_type, category, min_severity: info/warning/critical, limit); get_source_health; get_coverage_audit
- **Резидентство** _(section)_
  - Элементы: get_client_status; get_checklist; get_deadlines; get_client_documents; list_clients; create_client; update_client_stage; get_templates
- **Документы клиента** _(section)_
  - Элементы: draft_document (params: client_id, document_type: application/project_description/report/extension_request/exit_notice/ird_description) — LLM-черновик документа в Markdown; generate_document (params: template_id, client_id, variables key=value, inline:bool) — готовый файл PDF/DOCX; список шаблонов — list_document_templates; list_document_templates — список имён файлов шаблонов для generate_document
- **Агенты** _(section)_
  - Элементы: ask_consultant; validate_document (params: document_text, procedure_type); get_next_steps; subscribe_to_changes; check_eligibility
- **Навигация и страницы сайта** _(section)_
  - Элементы: get_navigation — где что находится во внутренних интерфейсах и как туда попасть; search_site_pages — поиск по страницам публичного сайта (sk.ru/dochub): заголовок, ИИ-описание, авто-теги (params: query, tags, limit), раздел, URL

## Telegram-бот для клиентов резидентства (Telegram)

- **Аудитория:** Клиенты-резиденты, менеджеры
- **Аутентификация:** Привязка по email: бот запрашивает email, затем связывает Telegram-аккаунт с записью клиента в системе.
- **Как попасть:** Найти бота в Telegram, нажать Start или написать /start. Email должен совпадать с email клиента в системе.

### Команда /start — Приветствие и привязка — `/start`

**Назначение:** Инициализация бота: запрос email для привязки аккаунта к записи клиента.  
**Как попасть:** Написать боту /start.

- **Приветственное сообщение** _(section)_
  - Элементы: «👋 Добро пожаловать в бота системы резидентства Сколково!»; запрос email с примером «Пример: ivan@example.com»; если уже авторизован: «👋 Здравствуйте! Вы уже авторизованы.» + повторный показ главного меню
- **Сообщения авторизации по email** _(section)_
  - Элементы: успех: «✅ Вы успешно авторизованы!» (Email, Client ID); ошибка: «❌ Не удалось авторизоваться…»
- **Главное меню (inline-кнопки)** _(nav)_
  - Элементы: 📊 Мой статус; ⏰ Дедлайны; 📄 Документы; ❓ Задать вопрос

### Команда /help — Справка — `/help`

**Назначение:** Список доступных команд бота.  
**Как попасть:** Написать /help.

- **Справка** _(section)_
  - Элементы: «📖 Справка по командам»; /start — начать работу и привязать аккаунт; /status — узнать текущую стадию резидентства; /deadlines — посмотреть ближайшие дедлайны; /docs — список ваших документов; /ask <вопрос> — задать вопрос консультанту; /help — эта справка; 💡 кнопки под сообщениями для навигации
