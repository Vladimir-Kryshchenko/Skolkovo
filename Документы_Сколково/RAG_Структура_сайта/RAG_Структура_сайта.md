# RAG: Полная структура сайта «База Сколково»

> Навигационная карта для ИИ-чат-бота: каждая страница, вкладка, блок и надпись.
> Сгенерировано из исходного кода пакетом `src/navindex` (источник истины — `Tree()`).
> Не редактируйте вручную: при изменении страниц правьте `src/navindex/tree.go` и пересоберите командой `navindex`.

## Карта интерфейсов

| Порт | Интерфейс | Для кого | Как попасть |
| :--- | :--- | :--- | :--- |
| :8092 | Личный кабинет клиента (Портал) | Клиенты-резиденты, компании, подающие заявку на резидентство Сколково | Открыть ссылку из письма (magic-link) или зайти на :8092 и запросить ссылку по email. |
| :8093 | Чат-виджет «Консультант Сколково» | Все пользователи сайта | Кнопка-пузырь «Консультант» внизу справа на любой странице, либо отдельная страница :8093/chat. |
| :8094 | Дашборд консультанта | Менеджеры резидентства | Открыть :8094/consultant/dashboard, ввести логин/пароль консультанта. |
| :8091 | Резидентство-Админ | Менеджеры резидентства | Открыть :8091/clients. |
| :8090 | Главная Админ-панель | Администраторы | Открыть :8090, войти логином/паролем. |
| :8080 | MCP-сервер (открытый API для ИИ-агентов) | ИИ-агенты, приложения, разработчики | POST :8080/mcp с заголовком Authorization: Bearer <ключ>. /health открыт. |
| Telegram | Telegram-бот для клиентов резидентства | Клиенты-резиденты, менеджеры | Найти бота в Telegram, нажать Start или написать /start. Email должен совпадать с email клиента в системе. |

## Личный кабинет клиента (Портал) (:8092)

- **Аудитория:** Клиенты-резиденты, компании, подающие заявку на резидентство Сколково
- **Аутентификация:** Magic-link на email (без пароля). Сессия 24 часа, токен ссылки 15 минут.
- **Как попасть:** Открыть ссылку из письма (magic-link) или зайти на :8092 и запросить ссылку по email.

### Вход в личный кабинет — `/login`

**Назначение:** Запросить ссылку для входа без пароля.  
**Как попасть:** Открывается автоматически, если не авторизован.

- **Форма входа** _(form)_ — Запрос magic-link по email.
  - Элементы: Логотип «🏛 База Сколково»; Личный кабинет резидента; Электронная почта (Email); поле name@company.ru; кнопка «Отправить ссылку для входа»; переключатель темы
- **Сообщения** _(section)_
  - Элементы: успех (зелёный); ошибка (красный)

### Дашборд — `/dashboard`

**Назначение:** Обзор текущей стадии, дедлайнов, прогресса и новостей базы.  
**Как попасть:** Пункт меню «Дашборд» (первый в навигации) или вход по ссылке.

- **Текущая стадия** _(section)_ — Стадия резидентства и прогресс по стадиям.
  - Элементы: бейдж стадии; progress bar; процент
- **Прогресс чек-листов** _(section)_
  - Элементы: название чек-листа; процент завершения; progress bar; пусто: «Чек-листы пока не назначены»
- **Ближайшие дедлайны** _(table)_
  - Элементы: Дедлайн; Дата; Статус; пусто: «Нет ближайших дедлайнов»
- **Статус документов** _(section)_
  - Элементы: бейджи pending/submitted/approved/rejected; пусто: «Документы пока не назначены»
- **Что нового в базе Сколково** _(widget)_ — Лента изменений базы: новые/обновлённые/устаревшие сущности.
  - Элементы: бейдж Новая/Обновлена/Устарела/Удалена; тип сущности; дата; заголовок; описание

### Мои чек-листы — `/checklists`

**Назначение:** Список шагов для прохождения процедур резидентства.  
**Как попасть:** Пункт меню «Чек-листы».

- **Карточки чек-листов** _(section)_
  - Элементы: название; тип процедуры; статус; progress bar; X/Y шагов; шаги: выполнен/в процессе/не начат; пусто: «Чек-листы пока не назначены»

### Дедлайны — `/deadlines`

**Назначение:** Сроки подачи документов и отчётности.  
**Как попасть:** Пункт меню «Дедлайны».

- **Ближайшие дедлайны** _(table)_
  - Элементы: Название; Дата; Тип; Статус; completed/overdue/upcoming
- **Просроченные** _(table)_
  - Элементы: Название; Дата; Статус; бейдж «Просрочен»; пусто: «Нет просроченных дедлайнов»

### Мои документы — `/documents`

**Назначение:** Документы, связанные с резидентством клиента.  
**Как попасть:** Пункт меню «Документы».

- **Таблица документов** _(table)_
  - Элементы: Документ; Роль; Статус; Дата; Источник; пусто: «Документы пока не назначены»

### Генерация документа — `/generate`

**Назначение:** Создать документ из шаблона по данным профиля (PDF/DOCX).  
**Как попасть:** Пункт меню «Генерация».

- **Генерация документа из шаблона** _(form)_
  - Элементы: Шаблон документа (выпадающий список); — Выберите шаблон —; кнопка «Сгенерировать»
- **Доступные шаблоны** _(section)_
  - Элементы: карточки шаблонов (название, тип, версия); пусто: «Шаблоны пока не добавлены»

### Уведомления — `/notifications`

**Назначение:** Inbox персональных уведомлений об изменениях документов, важных для стадии клиента.  
**Как попасть:** Иконка «🔔 Уведомления» / ссылка из дашборда.

- **Список уведомлений** _(section)_
  - Элементы: Непрочитанные: X из Y; бейдж Критично/Важно/Инфо; заголовок; дата; описание; ссылка на источник; кнопка «Отметить прочитанным»; пусто: «Пока нет уведомлений об изменениях»

## Чат-виджет «Консультант Сколково» (:8093)

- **Аудитория:** Все пользователи сайта
- **Аутентификация:** Публичный (без авторизации).
- **Как попасть:** Кнопка-пузырь «Консультант» внизу справа на любой странице, либо отдельная страница :8093/chat.

### Чат с консультантом (standalone) — `/chat`

**Назначение:** Полноэкранный чат с ИИ-консультантом по базе Сколково.  
**Как попасть:** Открыть :8093/chat.

- **Шапка** _(nav)_
  - Элементы: Консультант Сколково; переключатель темы
- **Область сообщений** _(section)_
  - Элементы: сообщения пользователя/ассистента; Markdown; индикатор «Печатает…»
- **Поле ввода** _(form)_
  - Элементы: Введите сообщение…; Enter — отправить, Shift+Enter — новая строка; кнопка «Отправить»

### Встраиваемый виджет — `/chat-widget.js`

**Назначение:** Плавающая кнопка и панель чата, встраивается на сторонний сайт через <script>.  
**Как попасть:** Подключить <script src=".../chat-widget.js" data-skolkovo-chat='{...}'>.

- **Плавающая кнопка** _(button)_
  - Элементы: иконка чат-пузыря, низ-право
- **Панель чата** _(widget)_
  - Элементы: шапка «Консультант»; сообщения; поле «Сообщение…»; кнопка «Отправить»; кнопка закрытия

## Главная Админ-панель (:8090)

- **Аудитория:** Администраторы
- **Аутентификация:** HTTP Basic Auth (логин/пароль), роли: Администратор, Редактор, Наблюдатель.
- **Как попасть:** Открыть :8090, войти логином/паролем.

### Документы — `/`

**Назначение:** Каталог документов базы: статусы, категории, файлы, индексация; сборщик и валидатор.  
**Как попасть:** Пункт меню «Документы» (главная). Меню: Документы, Изменения, Сравнение, Аналитика, Граф, Льготы и НПА, Прокси, ИИ. Кнопки: Парсинг RSS, Индексация, Полный синк, Индекс структуры.

- **Вкладка «Документы»** _(tab)_
  - Элементы: поиск; фильтр статуса (Все/Действует/На проверке/Устарел/Архив/Отклонён); Заголовок; Категория; Статус; Файл; действия: смотреть/скачать/загрузить/версия/де-индекс/удалить
