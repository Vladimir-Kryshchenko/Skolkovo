# Правило проекта: схемы в Markdown оформляются через Mermaid

**Версия:** 1.0

**Дата:** 29.05.2026

**Область применения:** все Markdown-документы (`.md`) проекта «База Сколково» — аналитика, ТЗ, архитектура, спецификации, отчёты, бэклоги, протоколы.

---

## 1. Правило

Любая схема, диаграмма, блок-схема или процессная карта в `.md`-документах оформляется **только через Mermaid** в кодовом блоке с указанием языка:

````markdown
```mermaid
...тело диаграммы...
```
````

**ASCII-арт со стрелками, рамками и псевдографикой запрещён.**

---

## 2. Где какие типы Mermaid использовать

| Контент | Тип Mermaid | Когда использовать |
| :--- | :--- | :--- |
| Бизнес-процесс с ролями | `flowchart TD` + `subgraph` (по ролям) | BPMN-подобные процессы, user flow |
| Архитектура системы | `flowchart LR` или `flowchart TD` | Модули, сервисы, потоки данных |
| Последовательность взаимодействия | `sequenceDiagram` | API/MCP-вызовы, диалог сервисов, пайплайн RAG |
| Состояния объекта | `stateDiagram-v2` | Жизненный цикл документа (на проверке → действует → устарел) |
| Календарный план / Ганта | `gantt` | Спринты, этапы, дедлайны |
| Структура данных | `erDiagram` | ER-схемы БД, метаданные документов |

---

## 3. Что нельзя

- **Нельзя** заменять Mermaid на ASCII-арт со стрелками `→`, `↓`, рамками `┌─┐`.
- **Нельзя** оставлять схемы только в виде картинок (PNG/SVG) без Mermaid-исходника рядом.
- **Нельзя** использовать кодовый блок без указания языка `mermaid` — диаграмма не отрендерится.

---

## 4. Применение к существующим документам

При любом редактировании `.md` с ASCII-схемой: конвертировать в Mermaid, ASCII удалять полностью. Объёмную конвертацию фиксировать отдельной задачей в бэклоге.
