# Эпик A: Расширение источников данных

**Версия:** 1.0
**Дата:** 29.05.2026
**Статус:** В работе

## Цель

Расширить сбор данных за пределы документов dochub.sk.ru — охватить все ключевые источники Сколково: мероприятия, конкурсы, гранты, требования резидентства, FAQ, Telegram-каналы, реестр резидентов.

## Задачи

### A1. Парсер мероприятий (sk.ru/events)

- [ ] Исследовать структуру sk.ru/events — RSS/JSON/HTML
- [ ] Реализовать `src/events/` — парсер мероприятий
- [ ] Модель Event: название, дата, место, описание, ссылка, статус
- [ ] Индексация мероприятий в RAG как категория «Мероприятия»
- [ ] MCP-инструмент `search_events`
- [ ] Админка — вкладка мероприятий (список, статусы)
- [ ] Интеграция в pipeline (регулярный сбор)

### A2. Парсер конкурсов и грантов

- [ ] Исследовать sk.ru/contests и sk.ru/grants
- [ ] Реализовать `src/contests/` — парсер конкурсов/грантов
- [ ] Модель Contest: название, описание, даты начала/окончания, требования, призы, ссылка, статус
- [ ] Индексация в RAG как категория «Конкурсы»
- [ ] MCP-инструмент `search_contests`
- [ ] Админка — вкладка конкурсов с фильтрацией по дедлайну
- [ ] Авто-определение просроченных конкурсов

### A3. Парсер требований резидентства

- [ ] Исследовать dochub.sk.ru и sk.ru на страницы требований
- [ ] Выделить структуру: критерии входа, этапы, документы, сроки
- [ ] Сохранить как структурированные документы категории «Требования»
- [ ] Версионирование изменений требований
- [ ] MCP-инструмент `get_residency_requirements`

### A4. Парсер FAQ dochub.sk.ru

- [ ] Исследовать dochub.sk.ru/faq или аналог
- [ ] Реализовать `src/faq/` — парсер FAQ
- [ ] Модель FAQ: вопрос, ответ, категория, ссылка
- [ ] Индексация в RAG — основа для чат-бота
- [ ] MCP-инструмент `search_faq`

### A5. Мониторинг Telegram-каналов

- [ ] Определить каналы: @Skolkovo, @SkolkovoVenture и др.
- [ ] Реализовать `src/telegram/` — через Telegram API или парсинг
- [ ] Модель TelegramPost: канал, текст, дата, ссылки, медиа
- [ ] Индексация в RAG как категория «Telegram»
- [ ] Фильтрация по релевантности (только Сколково-темы)

### A6. Парсер реестра резидентов

- [ ] Исследовать реестр на sk.ru (если публичный)
- [ ] Реализовать `src/registry/` — парсер реестра
- [ ] Модель Resident: название, ИНН, направление, дата вступления, статус
- [ ] Индексация в RAG как категория «Реестр»
- [ ] MCP-инструмент `search_residents`

## Зависимости

- E1 (Парсер) — должен быть завершён
- E2 (RAG) — должен быть завершён
- E3 (MCP) — должен быть завершён

## Критерии приёмки

- Все 6 источников парсятся автоматически по расписанию
- Данные индексируются в RAG с корректными категориями
- MCP-инструменты для поиска по каждому типу данных
- Админка показывает данные из всех источников
- Отчёты парсинга включают метрики по каждому источнику
