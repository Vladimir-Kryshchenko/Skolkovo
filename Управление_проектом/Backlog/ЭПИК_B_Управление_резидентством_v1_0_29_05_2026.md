# Эпик B: Система управления резидентством

**Версия:** 1.0
**Дата:** 29.05.2026
**Статус:** В работе

## Цель

Создать подсистему управления клиентами (заявителями/резидентами) — отслеживание статуса, чек-листы процедур, дедлайны, генерация документов, личный кабинет.

## Задачи

### B1. Модель клиента

- [ ] DB-миграция: таблица `clients` (id, name, inn, contact_email, contact_phone, residency_stage, created_at, updated_at)
- [ ] Модель `Client` в `src/common/model/client.go`
- [ ] Store-интерфейс для клиентов (CRUD + поиск)
- [ ] Админка — раздел «Клиенты» (список, карточка, статус)
- [ ] MCP-инструмент `get_client`

### B2. Стадии резидентства

- [ ] Модель `ResidencyStage`: подача_заявки → экспертиза → решение → договор → резидент → отчётность → продление / выход
- [ ] Поле `residency_stage` в модели Client
- [ ] Переходы между стадиями с валидацией
- [ ] История изменений стадий (audit log)
- [ ] MCP-инструмент `get_client_status`

### B3. Чек-листы процедур

- [ ] Модель `Checklist`: id, title, procedure_type, steps (JSON), version
- [ ] Типы процедур: `entry` (вход), `reporting` (отчётность), `extension` (продление), `exit` (выход)
- [ ] Модель `ChecklistStep`: id, checklist_id, title, description, order, required_docs, deadline_days
- [ ] Модель `ClientChecklist`: client_id, checklist_id, status, started_at, completed_at
- [ ] Модель `ChecklistStepStatus`: step_id, status (pending/in_progress/done/skipped), completed_at, notes
- [ ] Стандартные чек-листы для каждой процедуры
- [ ] MCP-инструмент `get_checklist`
- [ ] Админка — управление чек-листами, просмотр прогресса клиента

### B4. Трекер дедлайнов

- [ ] Модель `Deadline`: id, client_id, title, due_date, type, status, notification_sent
- [ ] Типы: `reporting` (квартальная/годовая), `extension`, `application`, `document_submission`
- [ ] Автогенерация дедлайнов из чек-листов и стадии резидентства
- [ ] Планировщик проверки просроченных/ближайших дедлайнов
- [ ] Уведомления (webhook, email, telegram) о приближающихся дедлайнах
- [ ] MCP-инструмент `get_deadlines`
- [ ] Админка — дашборд дедлайнов (календарь, просроченные, ближайшие)

### B5. Генератор документов

- [ ] Система шаблонов: `src/templates/` (Go templates)
- [ ] Шаблоны: заявление на резидентство, квартальный отчёт, годовой отчёт, запрос на продление, уведомление о выходе
- [ ] Модель `DocumentTemplate`: id, name, type, template_file, variables (JSON), version
- [ ] Заполнение шаблона данными клиента → PDF/DOCX
- [ ] Сохранение сгенерированных документов в `Документы_Сколково/Сгенерированные/`
- [ ] MCP-инструмент `generate_document`
- [ ] Админка — генерация документов, история генераций

### B6. Связь документов с клиентом

- [ ] Модель `ClientDocument`: client_id, document_id (из БД документов), role (required/optional/submitted), status, submitted_at
- [ ] Автоматическое определение требуемых документов по стадии
- [ ] MCP-инструмент `get_client_documents`

### B7. Портал клиента (личный кабинет)

- [ ] `src/portal/` — веб-приложение (Go + HTMX или templ)
- [ ] Авторизация клиента (email + magic link / пароль)
- [ ] Дашборд: текущая стадия, прогресс чек-листа, ближайшие дедлайны
- [ ] Просмотр и скачивание документов
- [ ] Генерация документов из шаблонов
- [ ] Чат с ИИ-агентом (встроенный)
- [ ] История действий и уведомлений

## Зависимости

- E0 (Фундамент)
- E4 (Админка)
- A3 (Требования резидентства)
- D (MCP расширения)

## Критерии приёмки

- Клиент заведён в систему с полной информацией
- Чек-листы процедур ведут клиента от подачи до резидентства
- Дедлайны генерируются автоматически и отслеживаются
- Документы генерируются из шаблонов с данными клиента
- Портал клиента показывает актуальное состояние
- MCP-инструменты дают полный доступ к данным клиента

*Версия 1.0 от 29.05.2026.*
