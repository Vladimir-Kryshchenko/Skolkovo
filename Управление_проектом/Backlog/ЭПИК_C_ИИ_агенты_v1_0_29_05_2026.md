# Эпик C: ИИ-агенты и чат-бот

**Версия:** 1.0
**Дата:** 29.05.2026
**Статус:** В работе

## Цель

Создать ИИ-агентов для автоматизации консультаций, валидации документов, мониторинга изменений и сопровождения клиентов. Реализовать каналы доставки: Telegram-бот и web-виджет.

## Задачи

### C1. Агент «Консультант»

- [ ] `src/agents/consultant/` — агент ответов на вопросы
- [ ] Промпт: контекст из RAG, ограничения (только по документам Сколково)
- [ ] Цепочка: вопрос → RAG-поиск → формирование ответа с цитированием источников
- [ ] Поддержка уточняющих вопросов (multi-turn)
- [ ] MCP-инструмент `ask_consultant`
- [ ] Логирование вопросов и ответов для аналитики

### C2. Агент «Валидатор»

- [ ] `src/agents/validator/` — агент проверки документов
- [ ] Промпт: правила валидации из требований Сколково
- [ ] Вход: документ (текст/PDF/DOCX) + тип процедуры
- [ ] Выход: список замечаний (критичные/предупреждения/ок)
- [ ] Проверка полноты, формата, сроков, ссылок на нормативку
- [ ] MCP-инструмент `validate_document`
- [ ] Админка — просмотр результатов валидации

### C3. Агент «Монитор»

- [ ] `src/agents/monitor/` — агент проактивных уведомлений
- [ ] Подписка на изменения документов по категориям
- [ ] Анализ влияния: «это изменение касается вас потому что...»
- [ ] Генерация digest: «за неделю изменилось X, вам важно Y»
- [ ] MCP-инструмент `subscribe_to_changes`
- [ ] Интеграция с трекером дедлайнов (B4)

### C4. Агент «Онкоординатор»

- [ ] `src/agents/coordinator/` — агент ведения клиента по чек-листу
- [ ] Промпт: чек-лист клиента, текущая стадия, дедлайны
- [ ] Проактивные сообщения: «пора сделать X, осталось Y дней»
- [ ] Помощь с каждым шагом: «вот инструкция, вот шаблон»
- [ ] Эскалация: если клиент застрял — уведомление администратора
- [ ] MCP-инструмент `get_next_steps`

### C5. Telegram-бот

- [ ] `src/tgbot/` — Telegram Bot API
- [ ] Авторизация: привязка к клиенту по email/токену
- [ ] Команды:
  - `/start` — приветствие, привязка
  - `/status` — текущая стадия резидентства
  - `/deadlines` — ближайшие дедлайны
  - `/docs` — мои документы
  - `/ask <вопрос>` — вопрос агенту-консультанту
  - `/validate` — загрузить документ на проверку
  - `/help` — справка
- [ ] Inline-кнопки для навигации
- [ ] Отправка уведомлений о дедлайнах и изменениях
- [ ] Конфигурация: `TELEGRAM_BOT_TOKEN` в .env
- [ ] Docker-контейнер для бота

### C6. Web-виджет чата

- [ ] `src/chat_widget/` — встраиваемый JS-виджет
- [ ] UI: popup-чат с иконкой
- [ ] Подключение к MCP-серверу через Streamable HTTP
- [ ] Аутентификация: API-ключ или сессия клиента
- [ ] Поддержка Markdown в ответах
- [ ] Кастомизация: цвета, логотип, приветствие
- [ ] HTML-embed: `<script src="/chat-widget.js"></script>`
- [ ] Админка — настройка виджета

## Зависимости

- E2 (RAG) — для поиска документов
- E3 (MCP) — для инструментов
- B (Управление резидентством) — для контекста клиента
- D (MCP расширения) — для новых инструментов

## Критерии приёмки

- Агент-консультант отвечает на вопросы с цитированием источников
- Агент-валидатор проверяет документы по требованиям Сколково
- Агент-монитор генерирует digest изменений
- Агент-онкоординатор ведёт клиента по чек-листу
- Telegram-бот работает с полным набором команд
- Web-виджет встраивается на сайт и подключается к MCP
