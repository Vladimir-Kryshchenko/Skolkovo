# Эпик E: Качество данных и аналитика

**Версия:** 1.0
**Дата:** 29.05.2026
**Статус:** В работе

## Цель

Обеспечить качество данных: отслеживание связей между документами, сравнение версий, автоматическая классификация, аналитика использования, контроль полноты базы.

## Задачи

### E1. Граф связей документов

- [ ] Модель `DocumentLink`: source_id, target_id, link_type (references/supersedes/related), created_at
- [ ] Автоопределение связей: анализ текста на ссылки на другие документы
- [ ] Ручное назначение связей в админке
- [ ] MCP-инструмент `get_document_links(document_id)`
- [ ] Визуализация графа в админке (Mermaid/Sigma.js)

### E2. Diff-система сравнения версий

- [ ] `src/diff/` — модуль сравнения документов
- [ ] Сравнение по тексту: добавлено/удалено/изменено
- [ ] Сравнение по структуре: разделы, пункты
- [ ] HTML-вывод с подсветкой (как GitHub diff)
- [ ] MCP-инструмент `compare_documents(doc_id_v1, doc_id_v2)`
- [ ] Админка — вкладка «Сравнение версий»
- [ ] Автогенерация diff при обнаружении новой версии

### E3. ИИ-классификация документов

- [ ] `src/classifier/` — модуль автоклассификации
- [ ] Категории: документы_фонда, требования, процедуры, отчётность, новости, мероприятия, конкурсы, FAQ
- [ ] Классификатор на основе эмбеддингов + LLM (опционально)
- [ ] Авто-классификация при индексации
- [ ] Ручная корректировка в админке
- [ ] Метрика точности классификации

### E4. Дашборд аналитики

- [ ] `src/analytics/` — модуль аналитики
- [ ] Метрики:
  - Общее количество документов по категориям и статусам
  - Динамика изменений (новые/обновлённые/устаревшие за период)
  - Популярные запросы через MCP
  - Активность клиентов
  - Прогресс чек-листов
  - Дедлайны по статусам
- [ ] `src/admin/` — вкладка «Аналитика» с графиками (Chart.js)
- [ ] Экспорт отчётов (CSV, PDF)
- [ ] MCP-инструмент `get_analytics(period, metric)`

### E5. Валидатор полноты базы

- [ ] `src/validator/` — модуль проверки полноты
- [ ] Проверки:
  - Какие документы не пропарились (403/ошибки)
  - Какие документы без текста (пустые)
  - Какие документы без категории
  - Какие документы без связей (если должны быть)
  - Расхождения между RSS-каталогом и БД
- [ ] Отчёт в админке с рекомендациями
- [ ] MCP-инструмент `get_completeness_report`
- [ ] Авто-уведомление администратору при проблемах

## Зависимости

- E1 (Парсер)
- E2 (RAG)
- E4 (Админка)
- E3 (MCP)

## Критерии приёмки

- Граф связей отображает отношения между документами
- Diff показывает точные изменения между версиями
- Автоклассификация работает с точностью > 85%
- Дашборд аналитики показывает все ключевые метрики
- Валидатор полноты находит все проблемы базы
