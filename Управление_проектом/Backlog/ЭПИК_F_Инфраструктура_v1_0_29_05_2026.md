# Эпик F: Инфраструктура и надёжность

**Версия:** 1.0
**Дата:** 29.05.2026
**Статус:** В работе

## Цель

Обеспечить продакшен-готовность: TLS, бэкапы, миграции БД, CI/CD, мониторинг и алертинг.

## Задачи

### F1. TLS / Let's Encrypt

- [ ] `deploy/nginx/` — конфигурация nginx с Let's Encrypt
- [ ] Certbot в docker-compose.prod.yml
- [ ] Автообновление сертификатов (cron)
- [ ] HSTS, OCSP stapling, security headers
- [ ] HTTP → HTTPS redirect
- [ ] Тестирование SSL Labs (рейтинг A+)

### F2. Backup & Restore

- [ ] `scripts/backup.sh` — бэкап PostgreSQL + Qdrant + документов
- [ ] Расписание: ежедневный полный + ежечасный инкрементальный
- [ ] Хранение: локально + удалённо (S3/объектное хранилище)
- [ ] `scripts/restore.sh` — восстановление из бэкапа
- [ ] Тестирование восстановления (раз в месяц)
- [ ] Уведомление о失败ных бэкапах

### F3. Миграции БД

- [ ] `deploy/migrations/` — SQL-миграции с нумерацией
- [ ] `src/migrate/` — Go-модуль применения миграций
- [ ] Команда `skolkovo migrate` — применить все pending миграции
- [ ] Таблица `schema_migrations`: version, applied_at, checksum
- [ ] Откат миграций (down)
- [ ] Интеграция с CI (проверка миграций)

### F4. CI/CD Pipeline

- [ ] `.github/workflows/ci.yml` — расширенный CI:
  - go vet, go fmt, staticcheck
  - go test ./... с покрытием
  - go build (linux/amd64, linux/arm64)
  - Docker build и push
- [ ] `.github/workflows/deploy.yml` — деплой:
  - При push в main → deploy на staging
  - При tag → deploy на production
- [ ] `.github/workflows/security.yml` — security scan:
  - govulncheck
  - Trivy Docker scan
  - Dependabot config
- [ ] Pre-commit hooks: golangci-lint, go fmt

### F5. Метрики и мониторинг

- [ ] Prometheus-метрики в Go-сервисе:
  - HTTP request duration, count, status codes
  - MCP tool usage (count, duration, errors)
  - RAG search latency
  - Scraping duration, document count
  - Queue sizes (scheduler, notifications)
- [ ] `/metrics` endpoint для Prometheus
- [ ] `deploy/prometheus.yml` — конфигурация
- [ ] `deploy/grafana/` — дашборды Grafana:
  - Обзор сервиса
  - MCP-инструменты
  - RAG-поиск
  - Парсинг
  - Клиенты и дедлайны
- [ ] Алертинг (Alertmanager):
  - Сервис недоступен
  - Парсинг failed
  - Ошибки >阈值
  - Qdrant/Postgres down

### F6. Логирование

- [ ] Структурированное логирование (JSON)
- [ ] Уровни: DEBUG, INFO, WARN, ERROR
- [ ] Корреляция запросов (request_id)
- [ ] Лог-ротация (размер + время)
- [ ] Опционально: Loki + Grafana для агрегации

### F7. Health Checks

- [ ] `/health` — общая проверка (liveness)
- [ ] `/health/ready` — проверка готовности (readiness)
  - Qdrant доступен
  - PostgreSQL доступен
  - TEI доступен
- [ ] Health check в docker-compose
- [ ] Kubernetes probes (если K8s)

## Зависимости

- Все предыдущие эпики

## Критерии приёмки

- HTTPS работает с Let's Encrypt сертификатом
- Бэкапы создаются по расписанию и восстанавливаются
- Миграции БД применяются автоматически
- CI зелёный на каждом PR
- Grafana-дашборды показывают метрики
- Алерты приходят при проблемах
